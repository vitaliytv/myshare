## Огляд

Цей файл конфігурує Stryker для виконання тестів, використовуючи Vitest як тестовий раннер. Він налаштовує процес аналізу покриття для запуску лише тестів, що стосуються мутованих рядків, та визначає директорії для звітів. Конфігурація спирається на конфіги `mutation.json` та `incremental.json`, забезпечуючи збереження даних мутації та відстеження інкрементального виконання для оптимізації продуктивності прогонів.

## Поведінка

./relay/stryker.config.mjs

1. Визначає тестовий раннер як Vitest, посилаючись на конфігураційний файл Vitest.
2. Налаштовує аналіз покриття для запуску лише тих тестів, які покривають мутовану лінію, що підвищує швидкість порівняно з повним прогоном тестового набору.
3. Вказує тимчасову директорію для звітів Stryker.
4. Конфігурує репортери для виведення результатів у форматі JSON та чистий текст.
5. Створює файл `mutation.json` для збереження детальних даних мутації.
6. Встановлює режим інкрементального виконання, що дозволяє відновлювати результати між збоями, покращуючи продуктивність прогонів.
7. Вказує файл `incremental.json` для збереження даних інкрементального виконання.
