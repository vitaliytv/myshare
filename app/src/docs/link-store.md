## Огляд

Модуль працює з посиланнями через `links.json`, який є конфігураційною опорою для читання й оновлення списку. Публічні функції `listLinks`, `addLink`, `_resetForTest` дають змогу отримувати, додавати й скидувати стан цього списку для тестів. Модуль працює fail-safe: не кидає винятків назовні, а за певних помилок повертає порожнє значення, наприклад `null`.

## Поведінка

- `listLinks` — повертає збережений список посилань у порядку від найновішого до найстарішого; якщо сховище недоступне або дані нечитабельні, повертає порожній список чи доступний in-memory стан.
- `addLink` — додає URL на початок списку без дублювання; порожнє або не-рядкове значення не змінює стан і просто повертає поточний список; дані зберігаються у `links.json`, а за відсутності OPFS — у in-memory fallback.
- `_resetForTest` — скидає in-memory fallback між тестами.

## Публічний API

- listLinks — показує збережені посилання у зворотному порядку, щоб найновіші були першими
- addLink — додає URL на початок списку без дублікатів; порожні або не-рядкові значення ігнорує
- _resetForTest — скидає in-memory fallback між тестами, щоб ізолювати їхній стан

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
