## Огляд

Файл забезпечує переклад тексту українською через локальний OpenAI-compatible omlx API за адресою <http://127.0.0.1:8000/v1>. Він є read-only: не пише у ФС/БД, але звертається до мережі. Код спирається на `response.json`, використовує `OMLX_BASE_URL="http://127.0.0.1:8000/v1"` як базову адресу сервера та `DEFAULT_MODEL="gemma-4-e4b-it-OptiQ-4bit"` як типовий пріоритет моделі. `chunkText` ділить великий текст на частини для окремої обробки, `buildMessages` формує повідомлення для запиту, `extractContent` дістає текст перекладу з відповіді, `listOmlxModels` отримує список доступних моделей, `resolveModel` визначає модель для перекладу, `translateChunk` перекладає окрему частину, а `translateToUkrainian` координує переклад усього тексту. У межах одного прогону модуль використовує кешування, щоб повторно не виконувати зайві обчислення або мережеві звернення.

## Поведінка

- `OMLX_BASE_URL` — базова адреса локального OpenAI-compatible omlx API: `http://127.0.0.1:8000/v1`; задає ціль для мережевих запитів перекладу й списку моделей.
- `DEFAULT_MODEL` — рядок `gemma-4-e4b-it-OptiQ-4bit`; визначає бажану модель перекладу за замовчуванням, якщо вона доступна на сервері.
- `chunkText` — ділить великий текст на непорожні частини з урахуванням меж абзаців і рядків, щоб перекладати довгі субтитри послідовно без втрати структури.
- `buildMessages` — формує мінімальний набір chat-повідомлень, який просить модель повернути лише природний український переклад зі збереженням перенесень рядків.
- `extractContent` — дістає текст перекладу з `response.json` відповіді OpenAI-compatible `chat/completions` і повертає порожній рядок, якщо очікуваного вмісту немає.
- `listOmlxModels` — звертається мережею до локального omlx API за адресою `http://127.0.0.1:8000/v1` і повертає список ідентифікаторів завантажених моделей.
- `resolveModel` — обирає модель для роботи: віддає перевагу `DEFAULT_MODEL`, а якщо її немає серед завантажених, бере першу доступну.
- `translateChunk` — надсилає один фрагмент тексту на переклад до локального omlx через OpenAI-compatible endpoint і повертає очищений український результат з `response.json`.
- `translateToUkrainian` — перекладає весь текст українською послідовно по чанках, у межах прогону тримає стабільну ціль мережевих викликів і повертає підсумковий текст разом із парами «оригінал ↔ переклад» для порівняння.

## Публічний API

- OMLX_BASE_URL="<http://127.0.0.1:8000/v1>" — експортована базова адреса OMLX API, яку модуль використовує для мережевих звернень до локального сервера перекладу та отримання переліку моделей.
- DEFAULT_MODEL="gemma-4-e4b-it-OptiQ-4bit" — експортована резервна назва моделі, яку модуль використовує, коли запитана модель не задана або недоступна.
- Модуль працює read-only: не пише у ФС/БД, а взаємодіє лише з мережею та обробляє дані в пам’яті.
- Модуль використовує кешування у межах прогону, щоб повторно застосовувати вже отримані результати без зайвих мережевих запитів.
- chunkText — розбиває великий текст на послідовні фрагменти, щоб переклад уміщався в окремі запити й фінальний результат зберігав вихідний порядок.
- buildMessages — готує вміст запиту до chat API як набір повідомлень, який спрямовує модель на повернення саме українського перекладу в текстовому вигляді.
- extractContent — виділяє з відповіді API корисний текст перекладу та відкидає метадані відповіді, які не належать до фінального результату.
- listOmlxModels — звертається до <http://127.0.0.1:8000/v1>, щоб отримати доступні моделі для подальшого використання під час перекладу.
- resolveModel — визначає модель для перекладу, надаючи перевагу запитаній назві та переходячи на DEFAULT_MODEL, якщо потрібної моделі немає серед доступних.
- translateChunk — надсилає один фрагмент тексту до OMLX API, спирається на response.json і повертає український переклад цього фрагмента.
- translateToUkrainian — перекладає весь текст українською, поєднуючи розбиття на фрагменти, вибір моделі, мережеві запити, кешування в межах прогону та складання підсумкового тексту в початковому порядку.
- Приклад мережевого звернення модуля: запит до <http://127.0.0.1:8000/v1> для отримання списку моделей або надсилання фрагмента тексту на переклад через chat API.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Кешує результати в межах одного прогону.
