Компонент `App.vue` є головним інтерфейсом застосунку MyShare, що приймає посилання через механізм Android Share. Він керує відображенням історії прийнятих посилань та надає інструменти для роботи з ними, зокрема отримання субтитрів та переклад.

## Поведінка

1. **Ініціалізація**: При завантаженні застосунок отримує версію себе, завантажує збережені кеші мов субтитрів та перекладів, а також витягує історію попередньо прийнятих посилань.
2. **Прийом посилань**: Якщо це не Android-платформа, відображається поле для ручного введення URL для симуляції подією `Share`. На Android, застосунок автоматично обробляє вхідні дані з `Share intent`.
3. **Аналіз посилань**: Для кожного прийнятого URL компонент виконує наступне:
    * Витягує метадані сторінки (заголовок, favicon), зберігаючи їх у внутрішньому кеші.
    * Якщо URL належить до YouTube, визначає його `videoId` та запитує список доступних мов субтитрів до Supadata, кешуючи результат для швидкого доступу.
4. **Відображення історії**: У списку відображуються всі прийняті посилання. Для YouTube відображається статус субтитрів (UA, EN або відсутні) та кнопки для взаємодії.
5. **Отримання субтитрів**: При натисканні на кнопку "Cубтитри" викликається запит на отримання транскрипту для заданого `videoId` у пріоритетних мовах (UA > EN). Результат відображається у модалці.
6. **Переклад субтитрів**: Якщо доступні англійські субтитри (але не українські), при натисканні на "Перекласти" відбувається наступне:
    * Якщо переклад для цього відео ще не збережений, запитується англійський транскрипт.
    * Транскрипт передається до локальної MLX-моделі (якщо платформа Desktop) для перекладу в українську мову з відображенням прогресу.
    * Збережений переклад кешується.
7. **Управління моделлю**: На Desktop-платформі користувач може вибирати модель для роботи з субтитрами, що зберігається між сесіями.

## Гарантії поведінки

* Внутрішні кеші (метадані, мови, переклади) зберігаються локально, щоб мінімізувати повторні мережеві запити.
* Для YouTube субтитри завантажуються асинхронно: спочатку статус мов, а сам транскрипт — лише при виклику користувачем.
* Процес перекладу використовує багатоетапний підхід: спочатку отримує оригінал, потім обробляє його частково (чанками), відображаючи прогрес.
* Якщо платформа Android, функціонал ручного введення URL недоступний, а можливість роботи з MLX-моделями обмежена.
