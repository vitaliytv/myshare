## Огляд

Модуль надає публічні точки входу `extractTitle`, `findLinkByRel`, `extractFaviconUrl`, `resolveUrl`, `parseHtml` і `fetchPageMeta`, щоб отримувати короткі метадані сторінки для UI без запису стану. `fetchPageMeta` звертається до мережі й повертає дані для відображення сторінки, а решта функцій працює з HTML та URL: `parseHtml` готує вміст до читання, `extractTitle` дістає назву, `findLinkByRel` знаходить релевантне посилання, `resolveUrl` зводить адреси до абсолютного вигляду, а `extractFaviconUrl` визначає URL favicon і, коли явної іконки немає, формує стандартний `/favicon.ico`. Це дає змогу викликаючому коду або отримати готові метадані одним викликом, або окремо використати потрібні кроки розбору й нормалізації.

## Поведінка

- `extractTitle` дістає текст сторінкового `<title>` і повертає очищену назву або порожній рядок, щоб мати короткий підпис сторінки для UI.
- `findLinkByRel` знаходить у `<head>` перший `link` із потрібним `rel`-токеном, щоб перевірено вибирати релевантний ресурс сторінки.
- `extractFaviconUrl` визначає favicon сторінки за пріоритетними `link`-посиланнями або через стандартний fallback `/favicon.ico` і повертає абсолютний URL або порожній рядок, щоб UI міг показати іконку сторінки.
- `resolveUrl` перетворює абсолютне або відносне посилання на коректний абсолютний URL і повертає порожній рядок при невалідному значенні, щоб помилки адреси не ламали UI.
- `parseHtml` перетворює HTML-рядок на `Document`, щоб далі читати метадані сторінки через DOM.
- `fetchPageMeta` звертається до мережі, завантажує HTML сторінки, враховує redirects і повертає назву та favicon, а помилки віддає caller’у для обробки в інтерфейсі.

## Публічний API

- `extractTitle` — витягує назву сторінки з HTML-документа, щоб отримати текстовий заголовок сторінки.
- `findLinkByRel` — знаходить у документі `link`-елемент за значенням `rel`, щоб визначити адресу пов’язаного ресурсу, оголошеного в HTML.
- `extractFaviconUrl` — визначає URL favicon зі сторінки, щоб отримати адресу іконки сайту.
- `resolveUrl` — перетворює відносний URL на абсолютний URL відносно базової адреси сторінки.
- `parseHtml` — перетворює HTML на структуру документа, придатну для читання заголовка, посилань та інших метаданих сторінки.
- `fetchPageMeta` — звертається до мережі, завантажує HTML сторінки та повертає метадані, які вдається з неї витягти. Функція працює в режимі read-only і не записує дані у ФС або БД.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
