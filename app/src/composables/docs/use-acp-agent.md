## Огляд

Єдина публічна функція `useAcpAgent` запускає ACP-агентів у режимі read-only. Вона перехоплює помилки та не кидає винятків назовні. Конфіги, на які спирається код: settings.json.

## Поведінка

1. `useAcpAgent` надає застосунку єдиний шлюз для роботи з ACP-агентами, журналом взаємодій і погодженням дій.
2. Шлюз підключає доменний каталог операцій, щоб агент міг виконувати доступні функції myshare.
3. Для запуску агентів використовується домашній каталог користувача як безпечний робочий контекст; у середовищах без Tauri застосовується поточний каталог.
4. Доступні профілі Codex, Cursor і pi; вибір моделі pi визначається його `settings.json`, а не налаштуваннями застосунку.
5. Виконання операцій залишається read-only щодо файлової системи та бази даних.
6. Помилки обробляються у fail-safe режимі: зовнішній код не отримує винятків.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
