## Огляд

`useAgent` надає застосунку готовий спосіб працювати з агентом у межах `myshare`, не виходячи за межі читання даних.  

Гарантує лише поведінку без запису у ФС чи БД, а також не вводить окремого кешування.  

Для виклику використовується публічний інтерфейс `useAgent`.

## Поведінка

1. `useAgent` створює вбудований agent-gateway для `myshare` і віддає готовий набір можливостей для діалогу з користувачем: request/respond/approve, конфігурацію OMLX та journal.
2. Він підключає доменний каталог інструментів і системний prompt, щоб агент працював у межах предметної області застосунку.
3. Він маршрутизує виконання інструментів через `run`-transport, тобто виконує дії через JS-обробники інструментів, а не через Tauri commands.
4. Він не пише у ФС чи БД; робота з OPFS та subtitle/transcript потоком винесена в інші частини застосунку.
5. Він налаштовує ізоляцію даних агента через префікс сховища `myshare`, щоб усі пов’язані артефакти залишалися в межах цього застосунку.
