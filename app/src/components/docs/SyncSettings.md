## Огляд

Цей компонент реалізує діалогове вікно для конфігурації та управління процесом синхронізації додатку. Він дозволяє налаштувати URL-адресу ретранслятора, емітент Ory та Client ID, а також керувати статусом входу користувача.

## Поведінка

1. При моніторингу монтування компонента:
    * Компонент перевіряє збережену сесію, щоб встановити початковий стан входу користувача та заповнити поля конфігурації, якщо сесія присутня.
    * Компонент активує слухача для очікування зворотного виклику OAuth.
2. При зміні вхідних даних (Relay URL, Ory Issuer, Client ID):
    * Статус підключення відображається відповідно до поточної конфігурації та стану входу.
3. При натисканні кнопки "Увійти через Ory" (якщо користувач не увійшов):
    * Відправляється запит на ініціалізацію входу, використовуючи вказані URL та ідентифікатори.
    * Під час запиту відображається стан завантаження.
    * Після завершення спроби входу, стан оновлюється; при помилці відображається повідомлення.
4. Після успішного отримання зворотного виклику OAuth (через `listenForOAuthCallback`):
    * Проводиться оновлення стану з сесії (`refreshFromSession`).
    * Якщо це Android-платформа, виконується одноразове отримання даних (`pullOnce`).
    * Якщо це інша платформа, запускається постійна синхронізація (`startSync`).
5. При натисканні кнопки "Вийти":
    * Зупиняється поточний процес синхронізації (`stopSync`).
    * Сесія користувача очищається з сховища (`clearSession`).
    * Статус входу встановлюється як неавторизований.

## Гарантії поведінки

* Компонент відображає відповідний статус (увійшли/не увійшли) на основі даних, збережених у сесії або отриманих під час входу.
* Кнопка "Увійти через Ory" є активною лише у випадку, якщо всі обов'язкові поля конфігурації (Relay URL, Ory Issuer) заповнені.
* Успішний вхід користувача автоматично ініціює відповідний механізм синхронізації відповідно до платформи.
