## ADR Вибір Ollama-моделі для перекладу субтитрів

## Context and Problem Statement
Застосунок myshare додав переклад EN→UA субтитрів через локальний Ollama. Для повторних перекладів того самого відео використовується кеш у `localStorage`, але вибір конкретної LLM-моделі був жорстко закодований як константа. Потрібно: 1) дати користувачу можливість перемикати модель з UI; 2) зберігати вибір між сесіями.

## Considered Options
* `q-select` у тулбарі заголовка — список завантажених моделей Ollama, синхронізується з `localStorage` через `model-pref.js`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "`q-select` у `q-header` тулбарі", because список моделей завантажується через вже наявний `listOllamaModels` (`GET /api/tags`), вибір зберігається ключем `myshare.ollamaModel` у `localStorage` через новий модуль `model-pref.js`, а при відкритті перекладу передається у `translateToUkrainian({ model: selectedModel.value, … })`.

### Consequences
* Good, because transcript фіксує очікувану користь: користувач бачить усі завантажені Ollama-моделі й може переключитися без перезапуску застосунку.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `app/src/model-pref.js` — новий модуль: `STORAGE_KEY = 'myshare.ollamaModel'`, `loadModelPref(storage)`, `saveModelPref(model, storage)`.
- `app/src/App.vue` — `ollamaModels = ref([])`, `selectedModel = ref(DEFAULT_MODEL)`, `onMounted` fetch через `listOllamaModels()`, `<q-select>` у `<q-header>`.
- Дефолт-модель змінено з `gemma3:4b` на `aya-expanse:8b` (результат бенчмарку: 85 tok/s, peak RAM 5.8 GB, найприродніша українська серед 9 протестованих моделей).

## ADR Вибір дефолтної моделі перекладу субтитрів (aya-expanse:8b)

## Context and Problem Statement
Для перекладу EN→UA субтитрів через Ollama потрібно обрати дефолтну LLM-модель. Перша версія використовувала `gemma3:4b` (~3.3 GB), але не проводилось жодного порівняльного тестування якості перекладу.

## Considered Options
* gemma3:1b, gemma3:4b, gemma4:e2b, gemma4:e4b, qwen3:1.7b, qwen3:4b, mistral:7b, ministral-3:3b, ministral-3:8b, aya-expanse:8b, aya:8b, command-r7b

## Decision Outcome
Chosen option: "`aya-expanse:8b`", because бенчмарк (той самий EN-текст субтитрів через всі моделі однаковим промптом) показав: 85 tok/s, load 1.3s, total 6.8s, peak RAM 5.8 GB — і найкраща якість українського перекладу серед усіх протестованих. Aya Expanse є multilingual-спеціалізованою моделлю Cohere (23 мови, включно з українською).

### Consequences
* Good, because transcript фіксує очікувану користь: ~3× швидша від gemma4:e4b (еталону), займає 5.8 GB RAM замість 10.4 GB — не спричиняє swap на 16 GB Mac.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Бенчмарк: `/tmp/bench.py` (locally), тест-дані — субтитри `dQw4w9WgXcQ` (2089 символів), параметри `temperature=0.2`, `num_ctx=8192`.
- Рядок дефолту: `app/src/ollama.js` → `export const DEFAULT_MODEL = 'aya-expanse:8b'`.
- Відхилені через зациклення: gemma3:1b, qwen3:1.7b. Відхилено через reasoning без можливості відключення: qwen3:4b. Відхилено через тихий запуск — низька якість: mistral:7b.
- Дані RAM виміряні через `GET /api/ps` після завантаження моделі.
