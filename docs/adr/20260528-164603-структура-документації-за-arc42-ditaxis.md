## ADR Зберігання прийнятих посилань як масив у localStorage

## Context and Problem Statement
Застосунок `myshare` приймає URL через Android Share intent, але нічого не зберігав між запусками. Потрібно було реалізувати персистентне збереження посилань у локальному сховищі пристрою.

## Considered Options
* Зберігати лише останнє посилання — перезаписувати ключ `myshare.sharedText` при кожному прийомі
* Зберігати масив усіх прийнятих посилань у новому ключі `myshare.sharedUrls`

## Decision Outcome
Chosen option: "Зберігати масив усіх прийнятих посилань у новому ключі `myshare.sharedUrls`", because користувач явно обрав варіант 2 («Історія посилань») після того, як асистент представив обидва варіанти і описав різні API та UI-наслідки.

### Consequences
* Good, because transcript фіксує очікувану користь: UI замість одного посилання показує `q-list` усієї прийнятої історії; новий запис додається на початок масиву (індекс 0), що дає хронологічно відсортований список без додаткового сортування.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Реалізація: `app/src/url-history.js` — публічне API `loadUrlHistory(storage)` / `saveUrlHistory(storage, history)` / `appendUrlToHistory(storage, url)`. Ключ `localStorage`: `myshare.sharedUrls`. Модуль валідує malformed JSON, non-array і non-string entries. Тести: `app/src/url-history.test.js` (11 vitest-тестів, усі зелені). Інтеграція у `app/src/App.vue`: `onMounted` — читання, обробник події `myshare:android-share` — запис через `appendUrlToHistory`. Документація: `docs/explanation/components/url-history.md`, `docs/glossary.md` (записи `Local Storage`, `URL History`).
