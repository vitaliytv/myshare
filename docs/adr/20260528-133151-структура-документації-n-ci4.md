# Структура документації на основі n-ci4.mdc із міграцією README.md

**Status:** Accepted
**Date:** 2026-05-28

## Context and Problem Statement

Проєкт `myshare` мав лише `README.md` у корені репозиторію як єдине джерело документації. Правило `n-ci4.mdc` вимагає повноцінної Markdown-документаційної бази з Diátaxis-структурою, arc42, MADR v4 ADR, C4-нотацією та підтримкою autogen-зон. Потрібно побудувати цей каркас і перенести вміст `README.md` до нової структури.

## Considered Options

- Структура на основі `n-ci4.mdc` (arc42 + Diátaxis + MADR v4 + C4 + hybrid manual/autogen-зони + MkDocs Material viewer)
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Структура на основі `n-ci4.mdc`", because правило `n-ci4.mdc` є обов'язковим стандартом документації у цьому репозиторії; користувач явно задав його як специфікацію; вміст `README.md` рознесено по відповідних розділах `docs/` без втрати контенту.

### Consequences

- Good, because документація структурована за Diátaxis-квадрантами (`explanation/`, `how-to/`, `reference/`, `snapshots/`), arc42-шаблон із C4 Mermaid-діаграмами закладено в `explanation/architecture.md`, а AUTOGEN-маркери з `hash="sha256:pending"` підготовлено до першої регенерації з accepted ADR.
- Bad, because `README.md` видалено — кореневий entry-point файл відсутній; агенти та CI-інструменти, що очікують `README.md` у корені, потребуватимуть адаптації.

## More Information

Створені файли:

- `docs/adr/index.md` — autogen-індекс accepted ADR
- `docs/explanation/overview.md` — місія + AUTOGEN Capability Map (колишній вміст `README.md`)
- `docs/explanation/architecture.md` — arc42 + C4 Context/Container (Mermaid) + collapsible engineer-розділ
- `docs/explanation/strategy.md` — MANUAL Wardley-плейсхолдер
- `docs/explanation/components/.gitkeep`
- `docs/how-to/run.md` — інструкція `bun install` / `bun run start`
- `docs/reference/.gitkeep`, `docs/snapshots/.gitkeep`
- `docs/glossary.md` — entry-point термінів для LLM-промптів
- `docs/.docgen/config.yaml` — версія `1`; визначає `adr.dir`, `adr.status_marker = "**Status:** Accepted"`, `projections.prompts_dir`, `projections.manifest`
- `docs/.docgen/manifest.json` — початковий стан `{"zones": {}, "adr_index": {}}`
- `docs/.docgen/prompts/projection.md` — системний промпт для LLM-регенерації проекцій

`README.md` видалено командою `rm`. Autogen-зони містять `hash="sha256:pending"` — реальні хеші проставить перша регенерація. Наявний генератор: `scripts/docs-regen/` (`cli.js`, `manifest.js`, `projection.js`, `triggers.js`). Правило-джерело: `.cursor/rules/n-ci4.mdc`.
