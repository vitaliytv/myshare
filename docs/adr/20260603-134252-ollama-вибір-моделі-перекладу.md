# Вибір Ollama-моделі для перекладу субтитрів EN→UA

**Status:** Accepted
**Date:** 2026-06-03

## Context and Problem Statement

Для перекладу англійських субтитрів YouTube-відео через локальний Ollama потрібно обрати модель, яка відповідає двом обмеженням: Mac M5 з 16 GB unified RAM (не спричиняти своп-трешинг) і прийнятна якість перекладу українською. Перший варіант `gemma4:e4b` (~9.6 GB у RAM) заповнював своп до 16 GB і повністю гальмував систему при відкритих IDE та браузері.

## Considered Options

- `gemma3:4b` (~3.3 GB на диску, 4.4 GB у RAM, 37 tok/s) — обраний перший дефолт
- `gemma4:e4b` (~9.6 GB у RAM, 28 tok/s) — найвища якість, але спричиняла своп-трешинг
- `gemma3:1b` (101 tok/s) — зациклювалась (нескінченний цикл повторів)
- `qwen3:1.7b`, `qwen3:4b` — reasoning-режим не вимикається через `think:false` в ollama; виводили англійські міркування замість перекладу
- `mistral:7b`, `ministral-3:3b`, `ministral-3:8b` — нижча якість або повільніший load
- `gemma4:e2b` — менша якість, ніж `gemma3:4b`

## Decision Outcome

Chosen option: "`gemma3:4b`", because бенчмарк на реальних субтитрах (Rick Astley `dQw4w9WgXcQ`, 2089 символів) показав другу за якістю природну українську мову після `gemma4:e4b`, але втричі менший footprint (4.4 GB vs 9.6 GB) — влазить у RAM без свопу навіть при відкритих IDE та браузері. Рішення оновлено до `aya-expanse:8b` після розширеного бенчмарку — дивись Update 2026-06-03 нижче.

### Consequences

- Good, because transcript фіксує очікувану користь: 37 tok/s, load 1.7 s, повний текст субтитрів за ~14 с; система не гальмує.
- Good, because `resolveModel` у `app/src/ollama.js` відкочується до першої наявної моделі, якщо `DEFAULT_MODEL` не встановлена — безпечний fallback.
- Bad, because `gemma4:e4b` дає вищу якість перекладу (точніші ідіоми); `gemma3:4b` має дрібні огріхи у відмінюванні дієслів.

## More Information

- Константа `DEFAULT_MODEL = 'gemma3:4b'` у `app/src/ollama.js` (пізніше змінена — дивись Update 2026-06-03).
- Бенчмарк: `/tmp/bench.py` на 9 моделях (`gemma3:1b`, `qwen3:1.7b`, `ministral-3:3b`, `qwen3:4b`, `gemma3:4b`, `mistral:7b`, `ministral-3:8b`, `gemma4:e2b`, `gemma4:e4b`). Параметри: `temperature=0.2`, `num_ctx=8192`, `keep_alive='5m'`, `think=false`.
- Відхилені через зациклення: `gemma3:1b`. Відхилено через reasoning без відключення: `qwen3:1.7b`, `qwen3:4b`. Відхилено через низьку якість: `mistral:7b`.
- Відсутні в реєстрі ollama: `ministral:8b`, `towerinstruct`, `alma` (MT-спеціалізовані NLLB/MADLAD — не підтримані llama.cpp).
- `ollama list` підтверджує установку: `gemma3:4b a2af6cc3eb7f 3.3 GB`.

## Update 2026-06-03

**Розширений бенчмарк і зміна дефолту на `aya-expanse:8b`:**

Після розширеного бенчмарку на субтитрах `dQw4w9WgXcQ` (2089 символів, `temperature=0.2`, `num_ctx=8192`) модель `aya-expanse:8b` показала найвищу якість серед усіх протестованих: 85 tok/s, load 1.3 s, total 6.8 s, peak RAM 5.8 GB. Aya Expanse — multilingual-спеціалізована модель Cohere (23 мови, включно з українською). Константа `DEFAULT_MODEL` у `app/src/ollama.js` змінена на `'aya-expanse:8b'`.

**UI вибору моделі:**

Додано перемикач моделі в UI без перезапуску застосунку:
- `app/src/model-pref.js` — `STORAGE_KEY = 'myshare.ollamaModel'`, функції `loadModelPref(storage)` та `saveModelPref(model, storage)`.
- `app/src/App.vue` — `ollamaModels = ref([])`, `selectedModel = ref(DEFAULT_MODEL)`; список завантажується в `onMounted` через `listOllamaModels()` (`GET /api/tags`); `<q-select>` у `<q-header>` тулбарі; вибір передається у `translateToUkrainian({ model: selectedModel.value, … })`.
