## ADR Перехід із Ollama на omlx для перекладу субтитрів

## Context and Problem Statement

Застосунок `myshare` використовував `app/src/ollama.js` для перекладу EN→UA субтитрів YouTube через `localhost:11434` (Ollama REST API). У паралельному проєкті `nitra/task` той самий сценарій вже реалізований через `omlx` — OpenAI-compatible MLX сервер на `127.0.0.1:8000/v1`. Щоб уніфікувати підхід до локального LLM, було вирішено мігрувати `myshare` з Ollama на omlx.

## Considered Options

* Залишити Ollama (`localhost:11434`, Ollama-специфічні поля `keep_alive`, `options.num_ctx`)
* Перейти на omlx (`127.0.0.1:8000/v1`, OpenAI-compatible `/v1/chat/completions` та `/v1/models`)

## Decision Outcome

Chosen option: "Перейти на omlx", because у проєкті `nitra/task` той самий протокол вже обраний і перевірений; MLX нативніший на Apple Silicon; OpenAI-compatible API стандартніший, ніж Ollama-специфічний.

### Consequences

* Good, because `app/src/ollama.js` замінено на `app/src/omlx.js` із тим самим публічним API (`translateToUkrainian`, `listOmlxModels`, `DEFAULT_MODEL`) — заміна прозора для `App.vue`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Видалено: `app/src/ollama.js`, `app/src/ollama.test.js`
- Додано: `app/src/omlx.js`, `app/src/omlx.test.js`
- `DEFAULT_MODEL = 'gemma-4-e4b-it-OptiQ-4bit'`
- `OMLX_BASE_URL = 'http://127.0.0.1:8000/v1'`
- `app/src-tauri/capabilities/default.json` — дозволи `localhost:11434/*` → `localhost:8000/*`, `127.0.0.1:11434/*` → `127.0.0.1:8000/*`
- `app/src/model-pref.js` — `STORAGE_KEY` змінено з `'myshare.ollamaModel'` на `'myshare.omlxModel'`

## ADR Зберігання історії прийнятих посилань у localStorage

## Context and Problem Statement

Застосунок `myshare` приймає посилання через Android Share intent і відображав лише одне останнє посилання в UI — без збереження між сесіями. Після зміни вимог знадобилося зберігати всю послідовність прийнятих URL у локальному сховищі пристрою для їх подальшого перегляду.

## Considered Options

* Зберігати лише останнє посилання в існуючому ключі `myshare.sharedText`
* Зберігати масив усіх посилань у новому ключі `myshare.sharedUrls` (найсвіжіший — індекс 0)

## Decision Outcome

Chosen option: "Зберігати масив усіх посилань у новому ключі `myshare.sharedUrls`", because користувач явно обрав варіант 2 — повну історію посилань.

### Consequences

* Good, because transcript фіксує очікувану користь: користувач бачить усі раніше прийняті URL у `q-list`, а не лише останнє.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Новий модуль: `app/src/url-history.js` — `loadUrlHistory`, `saveUrlHistory`, `appendUrlToHistory`, `STORAGE_KEY = 'myshare.sharedUrls'`
- Тести: `app/src/url-history.test.js` (11 vitest-тестів)
- `app/src/App.vue` — на `onMounted` читає історію; на подію `myshare:android-share` додає URL на початок масиву і зберігає; UI — `q-list` усіх URL замість одного поля
- Валідація: malformed JSON, non-array, non-string entries ігноруються
- Документація: `docs/explanation/components/url-history.md`, `docs/explanation/architecture.md`, `docs/glossary.md`

## ADR Структура документації `docs/` замість `README.md`

## Context and Problem Statement

Проєкт `myshare` мав єдиний `README.md` у корені репо. Відповідно до правила `n-ci4.mdc` документація повинна будуватися за гібридною моделлю Diátaxis + arc42 + ADR у каталозі `docs/` з AUTOGEN-зонами та ручними секціями.

## Considered Options

* Залишити плаский `README.md` у корені
* Перенести контент у `docs/` за структурою `n-ci4.mdc` і видалити `README.md`

## Decision Outcome

Chosen option: "Перенести контент у `docs/` за структурою `n-ci4.mdc` і видалити `README.md`", because правило `n-ci4.mdc` вимагає саме цю структуру; `README.md` видалено, його контент рознесено по відповідних файлах `docs/`.
