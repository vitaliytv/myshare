## ADR Перехід з Ollama на omlx (OpenAI-compatible MLX endpoint) для перекладу субтитрів

## Context and Problem Statement
`myshare` (desktop/Mac) перекладав субтитри YouTube через локальний Ollama-сервер (`localhost:11434`, Ollama-специфічний JSON API). ## Considered Options
* **omlx** (`http://127.0.0.1:8000/v1`) — OpenAI-compatible протокол, нативний для Apple Silicon MLX
* **Ollama** (`http://localhost:11434`) — Ollama-специфічний API (`keep_alive`, `options.num_ctx`)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "omlx", because реалізація в `nitra/task` вже використовує той самий протокол; omlx нативний для Apple Silicon MLX і покриває той самий OpenAI-compatible інтерфейс без Ollama-специфічних полів.

* Good, because transcript фіксує очікувану користь: `POST /v1/chat/completions` з `choices[0].message.content`; `GET /v1/models` замість Ollama `/api/tags`; прибрано `keep_alive` і `options.num_ctx`; 99/99 тестів і `cargo check` — зелені після міграції.
