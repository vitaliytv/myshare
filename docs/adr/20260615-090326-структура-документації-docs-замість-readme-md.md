## ADR Скрипт `android:install` для встановлення APK на підключений телефон

## Context and Problem Statement
`myshare` потрібно тестувати офлайн на фізичному Android-пристрої без постійного USB-з'єднання з Vite dev-сервером. Існуючий скрипт `android` запускає лише dev-режим із hot-reload.

## Considered Options
* Додати окремий скрипт у кореневий `package.json`, який збирає debug-APK і встановлює його через `adb install -r`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Окремий скрипт `android:install` у `package.json`", because це мінімальна зміна без нових залежностей — шлях до APK відомий із попереднього білда (`app/src-tauri/gen/android/app/build/outputs/apk/arm64/debug/app-arm64-debug.apk`), а `adb` вже є передумовою для `android`-скрипту.

### Consequences
* Good, because transcript фіксує очікувану користь: після `bun run android:install` `myshare` працює офлайн без USB-кабелю та dev-сервера.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Команда у `package.json`: `"android:install": "bun --cwd=app run tauri android build --apk --debug && adb install -r app/src-tauri/gen/android/app/build/outputs/apk/arm64/debug/app-arm64-debug.apk"`. Документацію оновлено: `docs/how-to/run.md` — розділ «Встановити свіжий APK на телефон для офлайн-використання» з фіксом `INSTALL_FAILED_UPDATE_INCOMPATIBLE` через `adb uninstall com.vitaliytv.myshare`.
