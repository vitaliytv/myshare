## ADR Вибір ollama-моделі для перекладу субтитрів

## Context and Problem Statement
У застосунку `myshare` реалізовано автоматичний переклад EN→UK субтитрів YouTube-відео через локальний Ollama. Потрібно було обрати дефолт-модель, яка відповідає двом обмеженням: 16 GB unified RAM (M5 Mac) і прийнятна якість перекладу українською.

## Considered Options
* `gemma4:e4b` (8.0B параметрів, ~9.6 GB у RAM)
* `gemma3:4b` (~3.3 GB у RAM)
* `gemma3:1b` (~0.8 GB у RAM)
* `qwen3:1.7b`, `qwen3:4b`
* `ministral-3:3b`, `ministral-3:8b`
* `mistral:7b`

## Decision Outcome
Chosen option: "`gemma3:4b`", because тест показав баланс якість/RAM: 3.3 GB на диску, 37 tok/s, повний текст субтитрів за ~14s, без свопу на 16 GB; якість трохи нижча за `gemma4:e4b`, але суттєво краща за решту. `gemma4:e4b` (9.6 GB) займає надто багато unified-пам'яті й при відкритому IDE/Ollama спричиняє своп-трешинг.

### Consequences
* Good, because `gemma3:4b` влазить у RAM без свопу (3.3 GB), а весь переклад субтитрів пісні — ~14 секунд.
* Bad, because `gemma3:4b` має дрібні граматичні огріхи (плутає особу дієслова); якщо якість стане критичнішою — треба буде перейти на `gemma4:e4b` або більшу multilingual-модель після розширення RAM.

## More Information
* Константа: `app/src/ollama.js` → `DEFAULT_MODEL = 'gemma3:4b'`
* Бенчмарк-скрипт: `/tmp/bench.py` (2089-символьний EN-текст, `num_ctx:8192`, `temperature:0.2`, `keep_alive:'5m'`, `think:false`)
* Модель-відмовники: `gemma3:1b` — цикли повторів; `qwen3:*` — reasoning не вимикається через `think:false` в ollama, не перекладає; `mistral:7b` — калька, повільний load
* Відсутні в реєстрі ollama: `ministral:8b`, `towerinstruct`, `alma` (MT-спеціалізовані NLLB/MADLAD — не підтримані llama.cpp)

## ADR Відображення статусу субтитрів через поле `availableLangs` supadata

## Context and Problem Statement
Застосунок `myshare` показував кнопку «Субтитри» для всіх YouTube-посилань, не знаючи заздалегідь, чи є субтитри українською або англійською. Фактична наявність мов з'ясовувалась лише після тапу (full-fetch транскрипту).

## Considered Options
* Окремий endpoint supadata для метаданих відео (список мов без тексту)
* Використання поля `availableLangs` з відповіді transcript-endpoint
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Використання поля `availableLangs` з відповіді transcript-endpoint", because supadata повертає `availableLangs` (повний список ISO 639-1 кодів) у **будь-якій** відповіді `GET /youtube/transcript`, тому один запит на відео дає список мов без окремого endpoint для метаданих.

### Consequences
* Good, because transcript фіксує очікувану користь: один HTTP-запит дає список всіх доступних мов, що дозволяє показати статус UA/EN/none одразу по кожному лінку.
* Bad, because кожна перевірка нового відео коштує один запит supadata (free tier — 100/місяць); тому список мов кешується в `localStorage` за `videoId` (`app/src/caption-langs.js`).

## More Information
* Endpoint: `GET https://api.supadata.ai/v1/youtube/transcript?videoId=…&text=true` (без параметра `lang` → повертає будь-яку доступну мову + `availableLangs`)
* Кеш: `app/src/caption-langs.js` → `loadLangsCache` / `saveLangsCache`, ключ `localStorage`: `yt-langs-cache`
* UI-статуси: 🇺🇦 UA / 🇬🇧 EN / «Без UA·EN» (tooltip з доступними мовами)
* Rust-команда: `yt_list_languages(video_id)` у `app/src-tauri/src/youtube.rs`, зареєстрована в `lib.rs`
* JS-обгортка: `getYoutubeLanguages(videoId)` у `app/src/youtube.js`

## ADR Переклад субтитрів через локальний Ollama (desktop-only)

## Context and Problem Statement
YouTube-відео без українських субтитрів, але з англійськими, раніше не перекладались. Виникла потреба: якщо статус `🇬🇧 EN` — дати кнопку перекладу EN→UK без зовнішнього API і без зайвих витрат.

## Considered Options
* Переклад через локальний Ollama (`http://localhost:11434`) безпосередньо з JS через `tauri-plugin-http`
* Нова Rust-команда для HTTP до Ollama
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Переклад через локальний Ollama напряму з JS через `tauri-plugin-http`", because Ollama — локальний сервіс (як `page-meta.js` вже робить HTTP до зовнішніх URL через той самий plugin), і нова Rust-команда була б надлишковою.

### Consequences
* Good, because переклад повністю локальний, безкоштовний, без зовнішнього API; результат кешується в `localStorage` за `videoId`, тож повторне відкриття — миттєво без запуску Ollama.
* Bad, because фіча доступна лише на desktop (Mac); на Android Ollama недоступний, тому кнопка «Перекласти» схована (`canTranslate = !isAndroidPlatform()`). Також потребує явного додавання `http://localhost:11434/*` у Tauri HTTP scope (`capabilities/default.json`) — без цього запити блокуються.

## More Information
* JS-модуль: `app/src/ollama.js` — `chunkText` (≤3500 символів по межах абзаців), `translateChunk` (`POST /api/chat`, `stream:false`, `temperature:0.2`, `num_ctx:8192`, `keep_alive:'5m'`), `resolveModel` (`GET /api/tags`)
* Кеш: `app/src/translation-cache.js`, ключ `localStorage`: `yt-translations`
* Tauri scope: `app/src-tauri/capabilities/default.json` — додано `http://localhost:11434/*`, `http://127.0.0.1:11434/*`
* UI: діалог `.cmp-grid` — двоколонкова сітка оригінал↔переклад, вирівняна посегментно
