# Статус субтитрів YouTube через `availableLangs` + localStorage-кеш

**Status:** Accepted
**Date:** 2026-06-02

## Context and Problem Statement

Застосунок `myshare` показував кнопку «Субтитри» для YouTube-лінків, але стан наявності мов (UA / EN / нема) ставав відомим лише після тапу та повного фетчу транскрипту. Потрібно показувати статус субтитрів одразу поруч із кожним лінком при завантаженні, не витрачаючи квоту supadata free tier (100 запитів/місяць) на повні транскрипти.

## Considered Options

- Виклик повного `yt_get_transcript` для кожного URL при рендерингу списку (витратить всю квоту)
- Нова команда `yt_list_languages` через один GET до supadata (поле `availableLangs` є в будь-якій відповіді transcript-endpoint) + кеш у `localStorage`

## Decision Outcome

Chosen option: "Нова команда `yt_list_languages` + localStorage-кеш", because supadata повертає `availableLangs` у будь-якій відповіді `/youtube/transcript`, тобто достатньо легкого запиту без завантаження тексту; кеш у `localStorage` (ключ `yt:langs`, ключ запису = `videoId`) гарантує, що для вже переглянутого відео API-запит більше не виконується і квота не витрачається.

### Consequences

- Good, because для вже побачених відео статус відображається миттєво з `localStorage`, без мережевого запиту до supadata; один легкий запит замість завантаження повного тексту зберігає квоту.
- Bad, because кеш не має TTL — потенційно застарілий статус після появи нових субтитрів, але це в transcript не обговорювалося.

## More Information

- Rust: `app/src-tauri/src/youtube.rs` — `yt_list_languages(video_id)`, зареєстрована в `app/src-tauri/src/lib.rs` через `tauri::generate_handler!`
- JS: `app/src/youtube.js` — `getYoutubeLanguages(videoId)` (обгортка над `yt_list_languages`)
- JS: `app/src/caption-langs.js` (новий файл) — `captionStatus(langs)` → `'uk' | 'en' | 'none'`; нормалізує `uk-UA`→`uk`, `en-US`→`en`; `loadLangsCache` / `saveLangsCache`
- UI: `app/src/App.vue` — `ensureYoutube` асинхронно запитує мови, кешує; чіп UA / EN / «Без UA·EN» + кнопка «↓ Субтитри» поруч із кожним YouTube-посиланням
- Тести: `app/src/caption-langs.test.js` (новий), `app/src/youtube.test.js` (розширено); 67 JS + 10 Rust — всі зелені

## Update 2026-06-03

Деталі реалізації відображення статусу субтитрів через `availableLangs`:

**Rust-команда:** `yt_list_languages(video_id)` у `app/src-tauri/src/youtube.rs`; зареєстрована в `app/src-tauri/src/lib.rs`.

**JS-обгортка:** `getYoutubeLanguages(videoId)` у `app/src/youtube.js`.

**Чиста логіка та кеш:** `app/src/caption-langs.js` — функція `captionStatus()` нормалізує `uk-UA`→`uk`, `en-US`→`en`; повертає `{ kind: 'uk'|'en'|'none', langs }`. Функції `loadLangsCache` / `saveLangsCache` кешують `videoId → availableLangs` у `localStorage` (ключ `STORAGE_KEY`).

**UI:** чіп 🇺🇦 UA / 🇬🇧 EN / «Без UA·EN» з tooltip повного переліку мов у `app/src/App.vue`.

Відео без субтитрів: supadata повертає 404 → порожній список `availableLangs`. Кожна перевірка нового `videoId` коштує 1 запит supadata (free tier — 100/місяць); повторні відкриття безкоштовні завдяки кешу. Реалізація перекладу EN→UA через локальний Ollama (desktop-only) — в `20260602-163928-ollama-переклад-субтитрів.md`.
