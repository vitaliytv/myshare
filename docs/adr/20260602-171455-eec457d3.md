## ADR

```

## ADR Ollama-переклад субтитрів EN→UA на десктопі (Mac)

## Context and Problem Statement
Коли відео має лише англійські субтитри, користувач не може читати їх українською. Потрібне локальне вирішення, яке не вимагає зовнішнього платного API і працює офлайн на Mac.

## Considered Options
* Ollama з локальною LLM-моделлю (прямий HTTP до localhost:11434 через tauri-plugin-http)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Ollama з локальною LLM-моделлю", because це дозволяє перекладати офлайн без зовнішніх витрат, модель (`gemma3:4b`) вміщується в 16 GB RAM без свопу (~3.3 GB проти ~9.6 GB у `gemma4:e4b`).

### Consequences
* Good, because transcript фіксує очікувану користь: переклад працює локально, квоту зовнішніх API не витрачає, результат кешується в localStorage (не перекладаємо двічі).
* Bad, because перша модель `gemma4:e4b` (9.6 GB) на 16 GB RAM спричиняла swap-трешинг і гальмування всього Mac — вирішено заміною на `gemma3:4b`.

## More Information
- `app/src/ollama.js`: `DEFAULT_MODEL = 'gemma3:4b'`, `POST http://localhost:11434/api/chat` (`stream: false`, `temperature: 0.2`, `num_ctx: 8192`, `keep_alive: '5m'`), `chunkText` (≤3500 символів по межах абзаців).
- `app/src/translation-cache.js`: кеш перекладів у `localStorage`.
- `app/src-tauri/capabilities/default.json`: додані `http://localhost:11434/*` і `http://127.0.0.1:11434/*` до HTTP-scope (без цього tauri-plugin-http блокував запити до Ollama).
- `app/src/App.vue`: кнопка «Перекласти» показується лише коли `canTranslate && status.kind === 'en'` (desktop + тільки EN субтитри), діалог `.cmp-grid` — двоколонкове порівняння оригінал↔переклад посегментно.
```
