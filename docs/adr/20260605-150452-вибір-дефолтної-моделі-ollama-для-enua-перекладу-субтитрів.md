## ADR Вибір дефолтної моделі Ollama для EN→UA перекладу субтитрів

## Context and Problem Statement
У застосунок myshare додали переклад англійських субтитрів українською через локальний Ollama. Модель `gemma4:e4b` (9.6 GB / ~10.6 GB RAM) стала причиною трешингу свопу на Mac з 16 GB: ваги займали майже весь unified-memory, і система переходила в активне свопування при завантаженні моделі.

## Considered Options
* `gemma4:e4b` — початкова дефолтна модель (8.0B параметрів, Q4_K_M, ~10.6 GB RAM)
* `gemma3:4b` — легша модель (Q4, ~4.4 GB RAM), присутня в реєстрі Ollama

## Decision Outcome
Chosen option: "`gemma3:4b`", because вона займає ~4.4 GB RAM (вдвічі менше за `gemma4:e4b`), не спричиняє своп-трешинг на 16 GB і дає якість перекладу, яка за benchmark-тестом виявилась лише незначно гіршою від `gemma4:e4b` при значно менших ресурсних вимогах.

### Consequences
* Good, because transcript фіксує очікувану користь: 4.4 GB RAM замість 10.6 GB, швидкість 37–38 tok/s, весь текст перекладається за ~14s без свопу навіть при відкритих Zed, VS Code та Claude.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Зміни в `app/src/ollama.js`: `DEFAULT_MODEL = 'gemma3:4b'`, `num_ctx: 8192`, `keep_alive: '5m'`. Benchmark-скрипт `/tmp/bench.py` порівнював 9 моделей: `gemma3:1b`, `qwen3:1.7b`, `ministral-3:3b`, `qwen3:4b`, `gemma3:4b`, `mistral:7b`, `ministral-3:8b`, `gemma4:e2b`, `gemma4:e4b`. За якістю перекладу перемогла `gemma4:e4b`, за балансом якість/RAM — `gemma3:4b`.

## ADR Налаштування Tauri HTTP scope для Ollama localhost:11434

## Context and Problem Statement
Кнопка «Перекласти» в Tauri-застосунку не могла достукатись до локального Ollama-сервера на `http://localhost:11434`, хоча інші запити (supadata, page-meta) проходили. Причина — патерн `http://**` у capabilities не матчить URL з явним портом.

## Considered Options
* Патерн `http://**` — початковий (не матчить явний порт)
* Явні записи `http://localhost:11434/*` та `http://127.0.0.1:11434/*` у capabilities

## Decision Outcome
Chosen option: "явні записи `http://localhost:11434/*` та `http://127.0.0.1:11434/*`", because wildcard `http://**` не покриває URL з явним портом в URLPattern Tauri, а явні записи гарантовано матчать потрібні адреси.

### Consequences
* Good, because transcript фіксує очікувану користь: після зміни `default.json` і перезапуску dev-застосунку переклад запрацював.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файл `app/src-tauri/capabilities/default.json`. Зміни: додано `{ "url": "http://localhost:11434/*" }` та `{ "url": "http://127.0.0.1:11434/*" }` у масив `allow` для `http:default`. Capabilities вшиваються в Rust-контекст під час збірки, тому зміна вступає в силу лише після перезапуску `bun run start`.

## ADR Налаштування keep_alive та num_ctx для Ollama-перекладу

## Context and Problem Statement
Після кожного перекладу модель ~9 GB лишалась у RAM ще 5 хвилин (дефолтний Ollama `keep_alive`), що підтримувало тиск на swap. Також дефолтний `num_ctx` gemma4 був надмірно великим, роздуваючи KV-кеш.

## Considered Options
* `keep_alive: 0` — негайне вивантаження після перекладу (тимчасово встановлено, потім відкликано)
* `keep_alive: '5m'` + `num_ctx: 8192` — явно обмежений контекст зі стандартним TTL

## Decision Outcome
Chosen option: "`keep_alive: '5m'` + `num_ctx: 8192`", because негайне вивантаження (`keep_alive: 0`) давало зайве навантаження при повторних перекладах (модель щоразу перезавантажується), а `num_ctx: 8192` обмежує KV-кеш до розміру, достатнього для чанків (≤3500 символів ≈ ~1200 токенів).

### Consequences
* Good, because transcript фіксує очікувану користь: модель не перезавантажується між чанками одного перекладу і при повторних запусках; KV-кеш менший.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Зміни в `app/src/ollama.js`. Константа `num_ctx: 8192` і `keep_alive: '5m'` передаються в тілі кожного `POST /api/chat`. Функція `unloadModel` (яку тимчасово додавали для `keep_alive: 0`) видалена. Тести: 93/93 ✅.

## ADR Вибір локальної моделі для агентного кодування через pi.dev

## Context and Problem Statement
Виникло питання, яка локальна Ollama-модель може виконувати агентні задачі (скіл `/n-fix`) через pi.dev на Mac з 16 GB. Провели benchmark по 7 tool-capable моделях: `qwen2.5-coder:7b`, `granite3.3:8b`, `llama3.1:8b`, `qwen3:8b`, `deepseek-coder-v2:16b`, `gemma4:e4b`, `gemma4:e2b`.

## Considered Options
* `qwen2.5-coder:7b` — coding-спеціалізована модель
* `qwen3:8b` — Qwen3 з підтримкою reasoning
* `llama3.1:8b` — Meta LLaMA 3.1
* `granite3.3:8b` — IBM, позиціонується як tool-орієнтована
* `deepseek-coder-v2:16b` — MoE coding-модель
* `gemma4:e4b` / `gemma4:e2b` — Google Gemma4
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "жодна локальна модель не обрана для агентного кодування", because тест через реальний `/skill:n-fix` у pi.dev показав, що жодна з 7 протестованих моделей не виконала задачу: `qwen2.5-coder:7b` і `granite3.3:8b` емітували tool-виклики як текст (0 виконань через Ollama OpenAI-compat shim); `deepseek-coder-v2:16b` — `does not support tools`; `gemma4:e4b` і `gemma4:e2b` — `does not support tools` для Gemma-родини; `llama3.1:8b` — 3 виклики, 9 помилок, нуль фіксів; `qwen3:8b` — частково (7/10 ❌ лишилось), timeout 600с, нові порушення.

### Consequences
* Good, because transcript фіксує очікувану користь: виявлено, що обмеження лежать на рівні Ollama OpenAI-compat шару (формат tool_calls), а не лише в якості моделей — це дає чіткий орієнтир для майбутніх спроб (нативний API замість OpenAI-shim).
* Bad, because для агентних скілів (`/n-fix`, `/n-coverage-fix`) локальні моделі непридатні — треба використовувати хмарні API (pi.dev підтримує `openai-codex/gpt-5.x`, Anthropic).

## More Information
Конфіг `~/.pi/agent/models.json` — Ollama підключено як OpenAI-сумісний провайдер (`baseUrl: http://localhost:11434/v1`, `api: openai-completions`). Тестовий стенд: повна rsync-копія репо з APFS-симлінком на `node_modules`, детермінований набір порушень (`yarn.lock`, `prettier` у devDeps, відсутній скрипт `lint-image`), watchdog 600с, реальний `/skill:n-fix`. RAM-метрики через `/api/ps`. Результати попереднього аналізу: `gemma3:4b` (`does not support tools`) теж непридатна для агентних задач, але відмінна для EN→UA перекладу.
