## ADR Зберігання прийнятих URL як масив у localStorage

## Context and Problem Statement
Застосунок `myshare` приймає посилання через Android Share intent і до цього лише відображав поточне посилання в UI без будь-якого збереження між запусками. Виникло питання, яку саме модель збереження обрати для прийнятих URL у локальному сховищі пристрою.

## Considered Options
* **Варіант 1** — зберігати лише останній прийнятий URL у наявному ключі `myshare.sharedText`
* **Варіант 2** — зберігати масив усіх прийнятих URL у новому ключі `myshare.sharedUrls`

## Decision Outcome
Chosen option: "Варіант 2 — масив у `myshare.sharedUrls`", because користувач явно обрав варіант 2 після того, як асистент сформулював обидва варіанти з різними API-наслідками.

### Consequences
* Good, because transcript фіксує очікувану користь: UI розширено до `q-list` із усіма прийнятими URL (найсвіжіший — індекс 0), що дає користувачеві доступ до повної історії між запусками застосунку.
* Bad, because transcript не містить підтверджених негативних наслідків.
