## ADR Індикатор наявності субтитрів через `availableLangs` без повного транскрипту

## Context and Problem Statement
У застосунку `myshare` список YouTube-посилань не показував, чи є субтитри для кожного відео, — це ставало відомо лише після явного тапу кнопки «Субтитри». Потрібно відображати статус (UA / EN / нема) одразу поруч із кожним лінком, не витрачаючи зайву квоту supadata.

## Considered Options
* Робити окремий запит повного транскрипту для кожного URL при завантаженні
* Використовувати поле `availableLangs` із відповіді transcript-endpoint без завантаження тексту (`text=true` без `lang`)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "використовувати `availableLangs`", because supadata повертає `availableLangs` у будь-якій відповіді `/youtube/transcript`, тож один запит на відео без фільтрації мови дає повний перелік доступних мов без завантаження тексту субтитрів.

### Consequences
* Good, because transcript фіксує очікувану користь: один легкий запит замість завантаження повного тексту зберігає квоту (free tier — 100 запитів/місяць).
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Нова Rust-команда: `yt_list_languages(video_id)` у `app/src-tauri/src/youtube.rs`
- Зареєстрована в `app/src-tauri/src/lib.rs` через `tauri::generate_handler!`
- JS-обгортка: `getYoutubeLanguages(videoId)` у `app/src/youtube.js`
- Чиста функція статусу: `captionStatus(langs)` у `app/src/caption-langs.js`; нормалізує `uk-UA`→`uk`, `en-US`→`en`
- Кеш мов у `localStorage` за `videoId`: `loadLangsCache` / `saveLangsCache`
- UI: чіп 🇺🇦 UA / 🇬🇧 EN / «Без UA·EN» поруч із кожним YouTube-посиланням у `app/src/App.vue`

## ADR Переклад субтитрів EN→UA через локальний Ollama (desktop-only)

## Context and Problem Statement
Коли для YouTube-відео є лише англійські субтитри, користувачеві потрібна можливість отримати їх у перекладі українською. Переклад має відбуватися локально (не через хмарний сервіс) і лише на десктопі (Mac), де Ollama доступний.

## Considered Options
* Хмарний переклад-сервіс (DeepL, Google Translate API тощо)
* Локальний Ollama через `POST /api/chat` зі стандартним model `gemma4:e4b`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Ollama локально", because на Mac вже встановлено Ollama з моделлю `gemma4:e4b`; локальний переклад не потребує хмарних API-ключів і не передає дані назовні.

### Consequences
* Good, because transcript фіксує очікувану користь: переклад без зовнішніх сервісів, модель підтверджено протестована через curl (`Hello…` → `Доброго дня…`).
* Bad, because фіча недоступна на Android (Ollama відсутній), — кнопка «Перекласти» показується лише коли `!isAndroidPlatform()`.

## More Information
- `app/src/ollama.js`: `translateToUkrainian({onProgress})`, `chunkText` (чанки ≤3500 символів по межах абзаців), `resolveModel` (дефолт `DEFAULT_MODEL = 'gemma4:e4b'`, fallback — перша з `/api/tags`)
- HTTP-виклики через `tauri-plugin-http` з JS напряму (без нової Rust-команди): `POST http://localhost:11434/api/chat`, `stream: false`, `temperature: 0.2`
- Системний промпт: `"You are a professional EN→UK subtitle translator. Translate the user text into natural Ukrainian..."`
- Перевірка готовності сервера: при помилці з'єднання підказка `ollama serve`

## ADR Кеш перекладів і мов субтитрів у `localStorage`

## Context and Problem Statement
Кожен запит до supadata (перевірка мов) і кожен запуск Ollama (переклад) коштують ресурси: supadata має квоту, Ollama — час і CPU. При повторному відкритті того самого відео необхідно уникнути повторних запитів.

## Considered Options
* Зберігати кеш у `localStorage`
* Зберігати кеш лише в пам'яті (reactive ref)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "`localStorage`", because `localStorage` переживає перезапуск застосунку і не потребує серверного сховища; `videoId` є стабільним ключем для обох типів кешу.

### Consequences
* Good, because transcript фіксує очікувану користь: повторне відкриття перекладеного відео відкриває результат миттєво без звернення до Ollama.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Кеш мов: `STORAGE_KEY` у `app/src/caption-langs.js`, `loadLangsCache` / `saveLangsCache`
- Кеш перекладів: `STORAGE_KEY` у `app/src/translation-cache.js`, `loadTranslations` / `saveTranslations`, запис містить `{ model, originalLang, segments: [{original, translated}] }`; `isValidEntry` відкидає биті записи
- Інвалідації кешу немає: transcript не фіксує рішення щодо TTL

## ADR Розширення Tauri HTTP-scope для `localhost:11434`

## Context and Problem Statement
Після реалізації Ollama-перекладу в застосунку виникла помилка: Tauri-capability `http:default` не пропускала запити до `http://localhost:11434`, бо wildcard `http://**` не матчив явний нестандартний порт за логікою urlpattern в `tauri-plugin-http`.

## Considered Options
* Додати явні записи `http://localhost:11434/**` і `http://127.0.0.1:11434/**` до `allow`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "явні записи для порту 11434", because wildcard `http://**` не покривав нестандартні порти; додавання точних URL-патернів — мінімальна зміна з гарантованим ефектом.

### Consequences
* Good, because transcript фіксує очікувану користь: Tauri дозволяє HTTP-виклики до Ollama API на `localhost:11434`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Файл: `app/src-tauri/capabilities/default.json`
- Додані записи: `{ "url": "http://localhost:11434/**" }` та `{ "url": "http://127.0.0.1:11434/**" }`
- Раніше наявні записи `http://**` і `https://**` збережені
