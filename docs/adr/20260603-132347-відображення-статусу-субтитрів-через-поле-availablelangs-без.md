## ADR Відображення статусу субтитрів через поле `availableLangs` без окремого API-ендпоінту

## Context and Problem Statement
Потрібно показувати по кожному YouTube-лінку наявність субтитрів (🇺🇦 UA / 🇬🇧 EN / нема) ще до тапу кнопки «Субтитри». Raніше статус з'ясовувався тільки після повного фетчу транскрипту.

## Considered Options
* Один запит до `GET /youtube/transcript` (без параметра `lang`) — читаємо поле `availableLangs` з відповіді.
* Окремий ендпоінт `/youtube/video` для метаданих.
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Один запит до `GET /youtube/transcript` з читанням `availableLangs`", because supadata повертає це поле в будь-якій відповіді transcript-endpoint, тобто один недорогий запит одночасно дає і список мов, і можливість дізнатись статус — без окремого ендпоінту.

### Consequences
* Good, because transcript фіксує очікувану користь: статус субтитрів видно одразу по кожному лінку без додаткових roundtrip-ів.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Нова Tauri-команда `yt_list_languages(video_id)` у `app/src-tauri/src/youtube.rs`; зареєстрована в `app/src-tauri/src/lib.rs`. JS-обгортка `getYoutubeLanguages(videoId)` у `app/src/youtube.js`. Чіп-статус у `app/src/App.vue`.

## ADR Переклад субтитрів EN→UA через локальний Ollama (desktop-only)

## Context and Problem Statement
Потрібно перекласти англійські субтитри в українські безпосередньо у застосунку, без залучення хмарних платних LLM-сервісів. Фіча актуальна для Mac/desktop.

## Considered Options
* Локальний Ollama (`http://localhost:11434`).
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Локальний Ollama", because на Mac Ollama встановлено (homebrew), є локальна модель, і переклад вже підтверджено живим тестом — не потребує квоти чи платежів.

### Consequences
* Good, because transcript фіксує очікувану користь: переклад працює офлайн, без API-ключів і без витрат на запит.
* Bad, because фіча доступна лише на desktop (`isAndroidPlatform()` gate), і залежить від того, чи запущено `ollama serve`.

## More Information
Модуль `app/src/ollama.js` — функції `chunkText` (ріже ≤3500 символів по межах абзаців), `translateChunk` (`POST /api/chat`, `stream:false`, `temperature:0.2`, `num_ctx:8192`, `keep_alive:'5m'`), `resolveModel`, `translateToUkrainian({onProgress})`. Перевірка платформи через `isAndroidPlatform()` у `app/src/App.vue`.

## ADR HTTP-запити до Ollama прямо з JS через `tauri-plugin-http` (без нової Rust-команди)

## Context and Problem Statement
Потрібно з Tauri-застосунку звертатись до `http://localhost:11434` (Ollama). Можна реалізувати через Rust-команду або безпосередньо з JS.

## Considered Options
* JS-запити через `@tauri-apps/plugin-http` (як у `page-meta.js`).
* Нова Rust Tauri-команда для Ollama.
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "JS-запити через `@tauri-apps/plugin-http`", because Ollama є локальним сервісом, а `tauri-plugin-http` вже використовується в проєкті за тим самим патерном (`page-meta.js`) — нова Rust-команда надлишкова.

### Consequences
* Good, because transcript фіксує очікувану користь: менше Rust-коду, рішення симетричне до наявного `page-meta.js`.
* Bad, because вимагало явно дозволити `localhost:11434` у Tauri HTTP scope (scope `http://**` не матчить явний порт).

## More Information
Виправлення scope у `app/src-tauri/capabilities/default.json`: додано `http://localhost:11434/*` і `http://127.0.0.1:11434/*`. Причина: Tauri URLPattern не матчить `http://**` проти URL із явним нестандартним портом.

## ADR Вибір `gemma3:4b` як дефолт-моделі Ollama замість `gemma4:e4b`

## Context and Problem Statement
Mac M5 з 16 GB unified RAM: `gemma4:e4b` завантажує ~9.6 GB ваг, що при наявних фонових процесах (VS Code, Zed, Claude) переповнює пам'ять, викликає `swap used = 16 GB` і повністю гальмує систему.

## Considered Options
* `gemma3:4b` (~3.3 GB на диску, 4.4 GB у RAM).
* `gemma4:e4b` (~9.6 GB у RAM) — вихідний дефолт.
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "`gemma3:4b`", because модель удвічі легша, підтверджена живим тестом (`ollama ps` показав 4.4 GB, 100% GPU, `num_ctx` та `keep_alive` застосувались коректно), і переклад якісний (`Hello…` → `Привіт усім…`).

### Consequences
* Good, because transcript фіксує очікувану користь: переклад без своп-трешингу на 16 GB RAM; `resolveModel` підстрахує, якщо модель не встановлена.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Константа `DEFAULT_MODEL = 'gemma3:4b'` у `app/src/ollama.js`. `resolveModel` відкочується на першу наявну модель, якщо дефолту немає. Перевірка `ollama ps` після живого тесту підтвердила параметри.

## ADR `keep_alive: '5m'` і `num_ctx: 8192` для Ollama-запитів

## Context and Problem Statement
Потрібно збалансувати два аспекти: не тримати модель у RAM надто довго (тиск пам'яті), але й не вивантажувати після кожного чанку (зайвий час перезавантаження між чанками одного перекладу).

## Considered Options
* `keep_alive: 0` — негайне вивантаження після перекладу.
* `keep_alive: '5m'` — стандартний Ollama-дефолт.
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "`keep_alive: '5m'`", because користувач явно попросив повернути 5 хв; `gemma3:4b` займає лише 4.4 GB, тому пам'яті вистачає, а модель не перевантажується між чанками одного перекладу та при повторному запуску.

### Consequences
* Good, because transcript фіксує очікувану користь: повторні переклади (наприклад, повторне відкриття діалогу) не чекають перезавантаження моделі.
* Bad, because модель займає ~4.4 GB упродовж 5 хв після останнього запиту.

## More Information
Параметр `num_ctx: 8192` доданий одночасно — обмежує KV-кеш замість дефолту моделі (131072 токенів у `gemma4`). Чанк ≤3500 символів ≈ ~1200 токенів, з відповіддю влазить у 8192 із запасом. Файл `app/src/ollama.js`, функція `translateChunk`.

## ADR Кеш готових перекладів субтитрів у `localStorage`

## Context and Problem Statement
Ollama-переклад однієї пісні / відео займає 15–30с. Повторний тап «Перекласти» для того самого відео не повинен запускати модель знову.

## Considered Options
* Зберігати `videoId → { model, segments }` у `localStorage`.
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Зберігати в `localStorage`", because це дозволяє відкривати переклад миттєво при повторному тапі без залучення Ollama.

### Consequences
* Good, because transcript фіксує очікувану користь: кнопка стає «Переклад» (вже перекладено), діалог відкривається без затримки.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Модуль `app/src/translation-cache.js` — `loadTranslations`, `saveTranslations`, `isValidEntry` (відкидає биті записи). Запис містить поля `model`, `originalLang`, `segments` (масив `{ original, translated }`).
