## ADR вибір Ollama-моделі для перекладу субтитрів EN→UA

## Context and Problem Statement
У застосунку `myshare` реалізовано переклад англійських субтитрів YouTube через локальний Ollama. Потрібно обрати дефолт-модель для перекладу, яка б давала прийнятну якість, не спричиняла своп-трешинг на MacBook з 16 GB unified memory та не вимагала явного керування пам'яттю після кожного перекладу.

## Considered Options
* `gemma4:e4b` (9.6 GB на диску, ~10.6 GB RAM) — найвища якість перекладу за результатами бенчмарку, але займає 10.6 GB unified RAM при `num_ctx 8192`, що на 16 GB машині неминуче викликає swap-трешинг за звичайного робочого навантаження.
* `gemma3:4b` (3.3 GB на диску, ~4.4 GB RAM) — дещо нижча якість, проте розміщується в пам'яті без свопу навіть за відкритого dev-оточення.
* `aya-expanse:8b`, `aya:8b`, `command-r7b` (5.1–6.8 GB RAM) — мультимовні LLM, спеціалізовані на перекладі; за результатами бенчмарку якість нижча за `gemma4:e4b` і не краща за `gemma3:4b`, при цьому споживання RAM вище.
* `gemma3:1b`, `qwen3:1.7b` — малі моделі; зациклювались або ігнорували інструкцію (reasoning-режим у `qwen3` не вимикається через `think:false` у llama.cpp/ollama).
* `mistral:7b`, `ministral-3:3b/8b` — прийнятна швидкість, але більш кальковий переклад і нижча точність.

## Decision Outcome
Chosen option: "`gemma3:4b`", because на 9-модельному бенчмарку (однаковий текст субтитрів Rick Astley, промпт EN→UK, `num_ctx 8192`, `temperature 0.2`) вона показала найкращий баланс: 37.5 tok/s, 4.4 GB RAM, повний текст без зациклень, природна українська — за умови, що апаратне обмеження (16 GB) є жорстким. Константа `DEFAULT_MODEL = 'gemma3:4b'` встановлена в `app/src/ollama.js`.

### Consequences
* Good, because `gemma3:4b` (~4.4 GB RAM) залишає ~11 GB вільними, тож переклад не викликає swap-трешингу навіть із відкритим dev-оточенням (VS Code, Zed, vite dev-server).
* Bad, because якість дещо нижча за `gemma4:e4b`: в бенчмарку `e4b` точніше відтворювала відмінки та дієвідміни. Різниця суб'єктивна для субтитрів, але помітна на уважному порівнянні.

## More Information

- RAM виміряно через `GET http://localhost:11434/api/ps` після першого inference (ваги + KV-кеш + буфери).
- `gemma4:e4b` видалено з ollama командою `ollama rm gemma4:e4b` після бенчмарку.
