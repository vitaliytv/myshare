## ADR Зберігання прийнятих URL як масив-історія у localStorage

## Context and Problem Statement

`myshare` приймає посилання через Android Share intent і відображав лише поточний URL. Постала задача зберігати прийняті посилання у локальному сховищі пристрою між запусками застосунку.

## Considered Options

- **Останнє посилання** — записувати у вже існуючий ключ `myshare.sharedText` одне значення; UI не змінюється.
- **Масив-історія** — зберігати всі прийняті URL у новому ключі `myshare.sharedUrls` як JSON-масив (найсвіжіший — індекс 0); UI розширюється `q-list`.

## Decision Outcome

Chosen option: "Масив-історія", because користувач явно обрав варіант 2 у відповідь на уточнення асистента.

### Consequences

- Good, because transcript фіксує очікувану користь: усі прийняті посилання зберігаються між сесіями; `url-history.js` ізолює логіку збереження від UI; 11 vitest-тестів покривають валідацію malformed JSON, non-array та non-string entries — усі 16 тестів (включно з `shared-url.test.js`) пройшли зелено.
- Bad, because масив зростає необмежено — transcript не містить підтвердженого механізму очищення чи обрізання історії.

## More Information

- Новий модуль: `app/src/url-history.js` — експортує `loadUrlHistory`, `saveUrlHistory`, `appendUrlToHistory`; ключ `STORAGE_KEY = 'myshare.sharedUrls'`.
- Тести: `app/src/url-history.test.js`.
- `app/src/App.vue` — `onMounted` завантажує історію; обробник події `myshare:android-share` викликає `appendUrlToHistory` і `saveUrlHistory`; шаблон замінено на `q-list`.
- Документацію компонента додано: `docs/explanation/components/url-history.md`.
- Glossary оновлено термінами `Local Storage` і `URL History` у `docs/glossary.md`.
