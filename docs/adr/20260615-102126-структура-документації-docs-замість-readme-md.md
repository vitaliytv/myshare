## ADR Перехід з Ollama на omlx для перекладу субтитрів

## Context and Problem Statement
Застосунок `myshare` використовував локальний Ollama-сервер (`http://localhost:11434`) для перекладу EN→UA субтитрів YouTube на desktop. Проєкт `nitra/task` вже перейшов на omlx (OpenAI-compatible MLX сервер), і виникла потреба узгодити стек.

## Considered Options
* Ollama API (`/api/chat`, порт 11434, `keep_alive`, `options.num_ctx`)
* omlx — OpenAI-compatible MLX сервер (`/v1/chat/completions`, порт 8000)

## Decision Outcome
Chosen option: "omlx — OpenAI-compatible MLX сервер", because користувач явно попросив перенести підхід з `nitra/task`, де Ollama вже замінено на omlx.

* Good, because transcript фіксує очікувану користь: той самий OpenAI-протокол (`choices[0].message.content`), що і в `nitra/task`; 99/99 vitest тестів проходять, `vite build` і `cargo check` — без помилок.
* Bad, because transcript не містить підтверджених негативних наслідків.
