## ADR Оптимізація розміру debug-APK для Tauri-застосунку

## Context and Problem Statement
Перша debug-збірка через `tauri android build --apk --debug` (без `--target`) створила universal APK розміром 775 MB. Він містив чотири копії `libmyshare_lib.so` (arm64-v8a, armeabi-v7a, x86, x86_64), кожна з повними debug-символами (~160 MB кожна). Пристрій розробника — сучасний arm64-телефон.

## Considered Options
* Збирати тільки під `aarch64` із `strip = true` у `[profile.dev]` в `Cargo.toml`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Збирати тільки під `aarch64` із `strip = true`", because телефон розробника — arm64, архітектури x86/x86_64/armeabi-v7a потрібні лише для емуляторів і старих пристроїв, а debug-символи роздмухують кожну `.so` з ~27 MB до ~160 MB без практичної цінності для ручного тестування.

### Consequences
* Good, because APK зменшився з 775 MB до 33 MB (×23 менше); всередині залишилась лише одна `lib/arm64-v8a/libmyshare_lib.so`.
* Bad, because збірка прив'язана до однієї архітектури; для запуску на x86-емуляторах потрібен окремий білд без `--target aarch64`.

## More Information
- `app/src-tauri/Cargo.toml` → додано `[profile.dev]\nstrip = true`
- Команда збірки: `bun --cwd=app run tauri android build --apk --debug --target aarch64`
- Артефакт: `app/src-tauri/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk`

## ADR npm-скрипти android:apk / android:install із копіюванням APK у корінь

## Context and Problem Statement
Скрипт `android:install` у кореневому `package.json` спочатку вказував на шлях `apk/arm64/debug/app-arm64-debug.apk`, тоді як Tauri реально збирав `apk/universal/debug/app-universal-debug.apk`. Додатково виникло бажання мати готовий `.apk` одразу в корені репо під зручним ім'ям без необхідності запам'ятовувати глибокий Gradle-шлях.

## Considered Options
* Оновити `android:install`, додати скрипт `android:apk` зі `--target aarch64` і `cp` у корінь, додати `myshare.apk` у `.gitignore`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Оновити `android:install`, додати `android:apk`", because потрібна одна команда для збірки та встановлення, а файл артефакту має бути доступний у корені репо без знання внутрішнього Gradle-шляху.

### Consequences
* Good, because `bun run android:apk` залишає `myshare.apk` у корені проєкту; `bun run android:install` будує і встановлює на телефон одним викликом; `android:install` делегує збірку в `android:apk`, тому прапорці оновлюються в одному місці.
* Bad, because `myshare.apk` — тимчасовий артефакт; доданий у `.gitignore`, але після `git clean` потребує повторної збірки.

## More Information
- `package.json` → `"android:apk"`: `bun --cwd=app run tauri android build --apk --debug --target aarch64 && cp app/src-tauri/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk myshare.apk`
- `package.json` → `"android:install"`: `bun run android:apk && adb install -r myshare.apk`
- `.gitignore` → додано рядок `myshare.apk`
